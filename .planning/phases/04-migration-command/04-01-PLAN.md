<objective>
Extend the data model and version-dispatch layer so that:
1. `UserConfig` gains an optional `hybrid_public` field that round-trips through `.sss.toml` with zero breakage for existing v1 repos.
2. `sss users add-hybrid-key <user> <b64>` writes a user's hybrid public key into the project config.
3. `resolve_suite_from_version` accepts `"2.0"` (currently it errors) and `load_project_config_internal` dispatches `open_repo_key` through `suite_for` so v2 repos are loadable.

Purpose: The `sss migrate` command (Plan 02) depends on all three of these changes. Without this wave, there is nowhere to store per-user hybrid public keys in `.sss.toml` and no way to load a v2 repo after migration.
Output: Modified src/project.rs, src/config.rs, src/commands/users.rs, src/commands/mod.rs, src/main.rs.
</objective>
