**Part B — Add `handle_keys_show`:**

Add after `handle_keygen_deprecated` (around line 465), before the randomart helper section. Gate the whole function with `#[allow(unused_variables)]` only if needed for non-hybrid builds:

```rust
fn handle_keys_show(main_matches: &ArgMatches) -> Result<()> {
    use libsodium_sys::{crypto_hash_sha256, crypto_hash_sha256_BYTES};

    let keystore = create_keystore(main_matches)?;

    // Load the raw StoredKeyPair — public keys only, no decryption needed.
    let stored = keystore.get_current_stored_raw().map_err(|_| {
        anyhow!("No current keypair found. Generate one with: sss keys generate --suite classic")
    })?;

    // --- Classic keypair block ---
    println!("Classic keypair:");
    let classic_key_bytes = stored.public_key.as_bytes();
    let mut classic_hash = vec![0u8; crypto_hash_sha256_BYTES as usize];
    unsafe {
        crypto_hash_sha256(
            classic_hash.as_mut_ptr(),
            classic_key_bytes.as_ptr(),
            classic_key_bytes.len() as u64,
        );
    }
    generate_randomart(&classic_hash, "SSS KEY (Classic)");

    // --- Hybrid keypair block (only if present) ---
    #[cfg(feature = "hybrid")]
    if let Some(ref hybrid_pk_b64) = stored.hybrid_public_key {
        println!();
        println!("Hybrid keypair:");
        let hybrid_key_bytes = hybrid_pk_b64.as_bytes();
        let mut hybrid_hash = vec![0u8; crypto_hash_sha256_BYTES as usize];
        unsafe {
            crypto_hash_sha256(
                hybrid_hash.as_mut_ptr(),
                hybrid_key_bytes.as_ptr(),
                hybrid_key_bytes.len() as u64,
            );
        }
        generate_randomart(&hybrid_hash, "SSS KEY (Hybrid)");
    }

    Ok(())
}
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI args → handler | `--suite` value from argv crosses into the keygen handler; clap validates the value_parser set |
| handler → keystore | Handler calls store_dual_keypair; hybrid secret material remains in Zeroizing wrapper until encryption (keystore layer concern from 03-01) |
| stored TOML → keys show | Public key base64 string read from disk and hashed; no secret material involved |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-06 | Elevation of Privilege | `--suite hybrid` / `--suite both` on non-hybrid build | mitigate | `#[cfg(not(feature = "hybrid"))]` guard in handler returns `Err("hybrid suite requires a --features hybrid build")` before any hybrid code path executes. Clap still parses the value; the runtime gate fires immediately. |
| T-03-07 | Tampering | `--suite both` on existing identity with hybrid already set | mitigate | `store_dual_keypair` Case B returns Err "hybrid keypair already present" when `stored.hybrid_public_key.is_some()`; handler propagates this error without overwriting. |
| T-03-08 | Spoofing | `--suite hybrid` silently auto-generates a classic key | mitigate | Handler checks `keystore.get_current_keypair(None)` before generating hybrid; if no classic key exists, errors with the exact actionable message from CONTEXT.md ("no classic keypair found; run `sss keygen --suite both`"). No silent auto-generation. |
| T-03-09 | Information Disclosure | `keys show` exposing secret material | accept | `handle_keys_show` calls `get_current_stored_raw()` which loads only the TOML (public fields); it never calls `load_keypair` or `decrypt_stored_keypair`. No passphrase is prompted; no secret bytes are read. |
| T-03-10 | Denial of Service | clap panics on unknown `--suite` value | accept | clap `value_parser(["classic", "hybrid", "both"])` rejects unknown values at arg parse time with a user-friendly error. The `_ =>` arm in the handler match is a belt-and-suspenders fallback. |
</threat_model>
