3. `sss-evil-toggle (beg end)` — docstring: "Evil operator to toggle encryption at point or region." `:motion evil-line`. Body:
   ```elisp
   (if (= beg end)
       (sss-toggle-at-point)
     (save-excursion
       (goto-char beg)
       (while (and (< (point) end)
                   (re-search-forward sss--any-marker-regexp end t))
         (goto-char (match-beginning 0))
         (sss-toggle-at-point)
         (forward-char 1))))
   ```
   When beg = end (no motion/count), toggle at point. When a region is given (`gt{motion}`), use `re-search-forward` with `sss--any-marker-regexp` to jump marker-to-marker. At each match, `(goto-char (match-beginning 0))` positions point at the marker start before calling `sss-toggle-at-point`, ensuring it is never called on inter-marker text (which would signal `user-error "No SSS marker at point"` and abort the operator). After toggling, `(forward-char 1)` advances past the match start so the next `re-search-forward` finds the next marker. This correctly decrypts sealed markers and encrypts open markers based on each marker's current state, rather than blindly encrypting everything.

   NOTE: `sss--any-marker-regexp` must already be defined earlier in the file. If it does not exist, define it before the evil block as a `defconst` matching both open (`SSS{...}`) and sealed (`SSS[...]`) marker prefixes, e.g.: `(defconst sss--any-marker-regexp "SSS[{\\[]" "Regexp matching the start of any SSS marker.")`

CRITICAL: Do NOT follow Pattern 5 in the research document (06-RESEARCH.md) — it uses `define-key evil-normal-state-map` (GLOBAL bindings). Use `evil-define-key 'normal sss-mode-map` as shown below (BUFFER-LOCAL bindings). Global bindings would clobber `ge`/`gd`/`gt` in ALL buffers.

Bind operators BUFFER-LOCALLY to sss-mode-map using `evil-define-key` (NOT `define-key evil-normal-state-map`). This avoids clobbering `ge` (evil-backward-word-end), `gd` (evil-goto-definition), and `gt` (evil-tab-next) globally in all buffers:

```elisp
(evil-define-key 'normal sss-mode-map
  (kbd "ge") #'sss-evil-encrypt
  (kbd "gd") #'sss-evil-decrypt
  (kbd "gt") #'sss-evil-toggle)
```

The research recommends buffer-local over global (Open Question #1). The ROADMAP says "In evil normal state, `ge{motion}` encrypts" without specifying global vs local. Buffer-local is strictly better UX: operators only shadow ge/gd/gt in sss-mode buffers where they are meaningful.

   Body:
   ```elisp
   (let ((bounds (sss--marker-at-point)))
     (when bounds
       (save-excursion
         (goto-char (car bounds))
         (when (re-search-forward "{" (cdr bounds) t)

   ```
