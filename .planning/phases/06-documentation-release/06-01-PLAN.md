### Algorithm Overview

State: the hybrid suite uses trelis (X448 + sntrup761) for key-encapsulation → BLAKE3 for KDF → libsodium XChaCha20-Poly1305 for AEAD. The in-file ciphertext layer is identical to the classic suite; only per-user .sss.toml entries differ.

Table:

| X448 public scalar | 56 | DH component |

| X448 secret scalar | 56 | DH component |
| sntrup761 secret key | 1763 | KEM component |

### KEM and Key-Derivation

Show the three-step process:
1. trelis `HybridKemKeypair::encapsulate(recipient_public_key)` produces `(encapsulation[1095], shared_bytes[64])`.
2. BLAKE3 `derive_key("sss hybrid kem v1", shared_bytes)` reduces to a 32-byte AEAD key.
3. libsodium XChaCha20-Poly1305 (`crypto_secretbox_easy` compatible) seals the 32-byte repo key `K` with a random 24-byte nonce.

### Sealed Key Wire Format

Show the layout as a struct diagram:

```
Hybrid sealed key (1167 bytes total):
  [0..1095)   encapsulation  — X448 ephemeral || sntrup761 ciphertext
  [1095..1119) nonce         — 24-byte random XChaCha20 nonce
  [1119..1167) ciphertext    — 32-byte K encrypted + 16-byte Poly1305 tag

Classic sealed key (80 bytes total, for comparison):
  [0..32)  ephemeral_pubkey  — X25519 ephemeral sender public key
  [32..80) ciphertext+mac    — 32-byte K encrypted + 16-byte XSalsa20-Poly1305 mac
```

Also state: per-user `.sss.toml` `sealed_key` field grows by approximately 1448 base64 characters per user entry (1167 bytes vs 80 bytes raw).

### Byte-Identical Ciphertexts Invariant

Explain the shared-K/shared-nonce invariant:
- The 32-byte repo key `K` is IDENTICAL regardless of which suite wrapped it.
- Nonce derivation (`BLAKE2b-192(key=K, input=timestamp||NUL||file_path||NUL||plaintext)`) uses the SAME `K` for both suites.
- Therefore the XChaCha20-Poly1305 AEAD ciphertext embedded in each `⊠{...}` marker is byte-for-byte identical whether `K` was wrapped via classic or hybrid.
- Migration (`sss migrate`) re-wraps the per-user entries in `.sss.toml` only; no file content is touched.

### Feature Gate

State: the hybrid suite is compiled only when the `hybrid` Cargo feature is enabled. The default build links only libsodium and contains no trelis code. Build with `cargo build --features hybrid` to include the hybrid suite.
