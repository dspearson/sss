Create docs/usage-guide.md covering the complete sss edit workflow. British English throughout. All CLI examples must be verified against src/main.rs before inclusion.

**CRITICAL accuracy rules from research:**
- Use `sss keys generate` (NOT `sss keygen` which is deprecated — handle_keygen_deprecated in main.rs)
- Use `sss seal --project` (NOT `sss process` which does not exist as a subcommand)
- Encryption is XChaCha20-Poly1305 via libsodium (NOT age crate, NOT scrypt)
- KDF is Argon2id (NOT scrypt)

**Sections to include:**

1. **Overview** — One paragraph: what sss does, marker-based approach
2. **Initial Setup**
   - Generate keys: `sss keys generate` (mention passphrase prompt, system keyring option)
   - Initialise project: `sss init <username>` (explain .sss.toml creation)
3. **Marking Secrets**
   - `⊕{secret}` (UTF-8) and `o+{secret}` (ASCII alternative)
   - Explain balanced brace counting — nested JSON is valid: `⊕{{"key":"value"}}`
   - Explain that surrounding text is preserved unchanged
4. **Sealing Files**
   - `sss seal <file>` (stdout), `sss seal -x <file>` (in-place)
   - `sss seal --project` (project-wide)
   - Explain transformation: `⊕{secret}` becomes `⊠{base64ciphertext}`
5. **Opening Files**
   - `sss open <file>` (stdout), `sss open -x <file>` (in-place)
   - `sss open --project` (project-wide, requires permission or SSS_PROJECT_OPEN=true)
   - Explain reverse transformation: `⊠{...}` becomes `⊕{decrypted}`
6. **Editing Files**
   - `sss edit <file>` — one-command workflow (decrypt, open $EDITOR, re-encrypt on save)
7. **Rendering**
   - `sss render <file>` — strips all markers, outputs pure plaintext
   - `sss render --project`
8. **Key Management**
   - `sss keys list` — list private keys
   - `sss keys pubkey` — export public key (mention --fingerprint)
   - `sss keys set-passphrase <key-id>` / `sss keys remove-passphrase <key-id>`
   - `sss keys current [key-id]` — show or set current keypair
   - `sss keys rotate` — project key rotation (explain it re-encrypts all files)
9. **Team Collaboration**
   - Bob generates keypair, exports pubkey: `sss keys pubkey > bob-key.txt`
   - Alice adds Bob: `sss project users add bob bob-key.txt`
   - Bob can now seal/open project files
   - Removing a user: `sss project users remove bob` (triggers automatic key rotation)
   - List users: `sss project users list`
10. **Git Integration**
    - `sss hooks install` — explain pre-commit (auto-seal), post-merge, post-checkout
    - `sss hooks install --multiplex` — .d/ directory structure
11. **Secrets Files**
    - `.secrets` file format: key-value pairs, multi-line with pipe syntax
    - `⊲{secret_name}` interpolation syntax
    - Custom filename/suffix via project config
12. **Stdin Support**
    - `echo "⊕{secret}" | sss seal -` — all commands support `-` for stdin

Cross-link to docs/configuration.md for detailed config reference.
