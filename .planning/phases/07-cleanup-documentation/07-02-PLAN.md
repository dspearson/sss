**Add new Section 8: Region Operations** (renumber subsequent sections). Document:
- `sss-encrypt-region` (`C-c C-e`): Select text, encrypt in-place. Auto-wraps plain text in open marker before sealing. Works on arbitrary text or existing open markers.
- `sss-decrypt-region` (`C-c C-d`): Select a sealed marker, decrypt in-place to open marker.
- `sss-toggle-at-point` (`C-c C-t`): Place point on any marker, toggles between encrypted and decrypted state. No need to select -- detects marker boundaries automatically.
- Include example workflow: type text, select, `C-c C-e` to encrypt, then `C-c C-t` to toggle back.

**Add new Section 9: Preview and Overlays.** Document:
- `sss-preview-at-point` (`C-c C-v`): Shows decrypted value in a transient overlay next to the sealed marker. Does NOT modify the buffer. Overlay dismisses on next keystroke.
- `sss-toggle-overlay-mode`: Enables/disables visual overlays on all markers. Sealed markers get `sss-sealed-face`, open markers get `sss-open-face`. Overlays include help-echo tooltips showing available commands.

**Add new Section 10: Auth-Source Integration.** Document:
- `sss-use-auth-source` custom variable (default t): When non-nil, sss-mode looks up the keystore passphrase from Emacs auth-source before each CLI call.
- How to set up: Add `machine sss login default password YOURPASSPHRASE` to `~/.authinfo` (or `~/.authinfo.gpg`).
- Benefit: Eliminates need for `SSS_PASSPHRASE` env var or system keyring. Works with all auth-source backends (file, gpg-encrypted file, secrets service).
- To disable: `(setq sss-use-auth-source nil)`.

**Add new Section 11: Command Menu (sss-dispatch).** Document:
- `sss-dispatch` (`C-c C-m`): Opens a command menu listing all SSS commands.
- When `transient` package is available (bundled with Emacs 28+, available on MELPA for 27.1): Shows a transient popup with categorised commands (Region Operations, Buffer/File, Project, Settings).
- When transient is unavailable: Falls back to `completing-read` with the same command list.
- This is the discoverability entry point -- new users can press `C-c C-m` to see all available commands.

**Add new Section 12: Evil Integration.** Document:
- Conditional loading: Evil features only load when evil-mode is present (`with-eval-after-load 'evil`). In vanilla Emacs, no evil code runs.
- Operators (normal state, sss-mode buffers only):
  - `ge{motion}` -- Encrypt text covered by motion (e.g., `geiw` encrypts word, `ge$` encrypts to end of line)
  - `gd{motion}` -- Decrypt text covered by motion
  - `gt{motion}` -- Toggle encryption state of markers in motion range
- Text objects:
  - `is` (inner sss) -- Select content inside marker braces (excluding `⊕{` / `⊠{` prefix and closing `}`)
  - `as` (outer sss) -- Select entire marker including prefix and braces
  - Usage: `vis` selects inner content, `das` deletes entire marker, `cis` changes inner content
- Note: `ge`, `gd`, `gt` bindings are buffer-local to sss-mode -- they do not override the default evil bindings (`evil-backward-word-end`, `evil-goto-definition`, `evil-tab-next`) in other buffers.

**Add new Section 13: Doom Emacs Integration.** Document:
- Conditional loading: Doom bindings only activate when `map!` macro is available (`(fboundp 'map!)`). Safe to load in non-Doom Emacs.
- Installation for Doom:
  1. Copy `emacs/sss-mode.el` to `~/.config/doom/lisp/sss-mode.el`
  2. Add to `~/.config/doom/config.el`: `(load! "lisp/sss-mode")`
  3. Run `doom sync`
- Leader bindings (`SPC e` prefix):
  - `SPC e e` -- Encrypt region
  - `SPC e d` -- Decrypt region
  - `SPC e t` -- Toggle at point
  - `SPC e v` -- Preview at point
  - `SPC e SPC` -- SSS menu (sss-dispatch)
  - `SPC e p i` -- Init project
  - `SPC e p p` -- Process project
  - `SPC e k g` -- Generate keys
  - `SPC e k l` -- List keys
- Localleader bindings (`, e` prefix, sss-mode buffers only):
  - `, e e` -- Encrypt region
  - `, e d` -- Decrypt region
  - `, e t` -- Toggle at point
  - `, e v` -- Preview at point
  - `, e SPC` -- SSS menu
