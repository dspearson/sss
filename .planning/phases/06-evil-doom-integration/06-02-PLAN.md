Wrap the entire Doom block in `(when (fboundp 'map!) ...)`. This guard ensures map! is defined before calling it — in vanilla Emacs, `map!` is void and this block is skipped entirely.

**Global leader bindings (DOOM-01):** `SPC e` prefix with nested sub-prefixes for project and keys:

```elisp
(map! :leader
      (:prefix-map ("e" . "encryption")
       :desc "Encrypt region"   "e" #'sss-encrypt-region
       :desc "Decrypt region"   "d" #'sss-decrypt-region
       :desc "Toggle at point"  "t" #'sss-toggle-at-point
       :desc "Preview at point" "v" #'sss-preview-at-point
       :desc "SSS menu"         "SPC" #'sss-dispatch
       (:prefix ("p" . "project")
        :desc "Init project"    "i" #'sss-init
        :desc "Process project" "p" #'sss-process)
       (:prefix ("k" . "keys")
        :desc "Generate keys"   "g" #'sss-keygen
        :desc "List keys"       "l" #'sss-keys-list)))
```

Key layout rationale:
- `SPC e e` — encrypt (mnemonic: "e" for encrypt)
- `SPC e d` — decrypt
- `SPC e t` — toggle
- `SPC e v` — preview (view)
- `SPC e SPC` — dispatch menu
- `SPC e p i` — project init
- `SPC e p p` — project process
- `SPC e k g` — keys generate
- `SPC e k l` — keys list

**Localleader bindings (DOOM-02):** `, e` prefix in sss-mode buffers only:

```elisp
(map! :localleader
      :map sss-mode-map
      (:prefix ("e" . "sss")
       :desc "Encrypt region"   "e" #'sss-encrypt-region
       :desc "Decrypt region"   "d" #'sss-decrypt-region
       :desc "Toggle at point"  "t" #'sss-toggle-at-point
       :desc "Preview at point" "v" #'sss-preview-at-point
       :desc "SSS menu"         "SPC" #'sss-dispatch))
```

The localleader is `, ` per user's config (`doom-localleader-key` = ","). The `:map sss-mode-map` ensures these only activate in sss-mode buffers.

Add `(declare-function map! "doom-core" t t)` BEFORE the `(when (fboundp 'map!) ...)` block. This silences the byte-compiler "free variable" / "unknown function" warning for `map!` without requiring Doom at compile time.

IMPORTANT:
- Do NOT call `(require 'sss)` or `(require 'sss-ui)` — these are old plugin packages being deleted in Phase 7. All target functions are already defined earlier in sss-mode.el.
- Do NOT use `(with-eval-after-load 'doom-core ...)` — `(when (fboundp 'map!) ...)` is simpler and achieves the same result.
- Do NOT use `(featurep 'evil)` inside the Doom block for operator bindings — the operators are already bound to ge/gd/gt in sss-mode-map by Plan 06-01's evil-define-key. Doom's map! for operators is unnecessary and would create global bindings we explicitly avoided.
- All `:desc` strings provide which-key descriptions for Doom's discoverable key hints.
